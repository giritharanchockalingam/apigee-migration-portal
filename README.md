# Apigee Migration Portal — Capella Canvas

A live demo portal for the MuleSoft → Apigee X integration migration. Visualizes
current-state vs. target-state architecture and a real end-to-end test of 33
endpoints running against actual Canvas LMS data through Apigee X on GCP.

## What's inside

```
apigee-migration-portal/
├── index.html        # Portal: architecture diagrams + live dashboard + approach
├── results.js        # Test results (regenerated by ../apigee-batch/test-all-33.sh)
├── vercel.json       # Vercel static-site config
└── README.md         # This file
```

Companion source (in the sibling directory `../apigee-batch/`):
- `migrate-all.js` — Node.js generator + deployer for all three Apigee proxies
- `test-all-33.sh` — end-to-end test, regenerates `results.js`
- `seed-canvas-data.sh` — populates local Canvas with rich demo data
- `bundles/` — generated Apigee proxy bundles

## Running locally

Open `index.html` directly in a browser. Chrome blocks `fetch()` from `file://`
origins, so the portal loads results via a `<script>` tag (`results.js`) which
works without a local server.

If you prefer a server:

```bash
cd apigee-migration-portal
python3 -m http.server 8000
# open http://localhost:8000
```

## Refreshing the live data

From `../apigee-batch/`:

```bash
bash test-all-33.sh
cp dashboard/results.js ../apigee-migration-portal/results.js
```

Then refresh the browser — the dashboard reflects the latest test run.

## Deployment

Auto-deploys to Vercel on push to `main`. See `vercel.json` for caching policy
(results.js is set to `no-cache` so refreshes show the latest test).

## Architecture summary

**Current state** — All 285 APIs on MuleSoft (single platform, single vendor).

**Future state** — Synchronous APIs (115) move to Apigee X on GCP for portfolio
rationalization. Async/orchestration/batch APIs (170) stay on MuleSoft or move
to Boomi in Phase 2.
